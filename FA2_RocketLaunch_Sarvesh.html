PI 1 · 2 Marks

Problem Analysis & DB
Requirement Identification

Identifying the core requirements, database needs, and design decisions for a web-based rocket launch management system.

The Model Rocket Launch Management System is a web-based application that extends the console-based rocket simulation to a full database-backed system. It allows users to manage rocket configurations, motor types, and flight records online. The system uses JSP for the frontend interface and MySQL for storing and managing launch data.
⚡ Functional Requirements
  • User login and role-based access (Admin / Operator)
  • Add, view, update, and delete motor specifications (A, B, C class)
  • Configure and register rocket models with assigned motors
  • Record launch events with altitude, velocity, and recovery status
  • View complete flight history with filtering by motor class
  • Admin panel to manage users and system data

PI 3 · 3 Marks

JSP GUI Design Concepts

Six JSP pages forming the complete web interface of the system, with a clear role for each page.

login.jsp
Accepts username and password. Validates credentials against the User table. Sets session attribute on success. Redirects to dashboard.
dashboard.jsp
Home page after login. Displays summary stats — total rockets, total launches, last apogee, motor stock. Protected by session check.
motor.jsp
Displays motor inventory table (A/B/C class). Allows adding new motors and updating stock quantity. Admin-only edit access.
rocket.jsp
Shows all registered rockets with motor assignments. Form to add or update rocket config. Linked to motor table via dropdown.
launch.jsp
Submit new launch form — select rocket, enter apogee, velocity, recovery status. Displays complete launch history table.
admin.jsp
Admin panel to manage all users, view all records, perform hard deletes, and reset launch data. Role-guarded page.

PI 4 · 2 Marks

Integration Logic
JSP–Database Flow

The complete request-response cycle — from user action on a JSP page all the way to database interaction and back.

01
→
JSP Page (Client)
User fills a form (e.g. Add Motor). Browser sends HTTP POST request with form data to the server.
02
→
Servlet / JSP Scriptlet
request.getParameter() extracts form values. Input validation is performed before any DB interaction.
03
→
JDBC Connection
DriverManager.getConnection() establishes a connection to the MySQL database using the DB URL and credentials.
04
→
SQL Execution
PreparedStatement executes INSERT / SELECT / UPDATE / DELETE query against the target table with bound parameters.
05
→
ResultSet Processing
rs.next() iterates through returned rows. Data is extracted from each column and stored in variables or beans.
06
→
JSP Response (Client)
Data is rendered dynamically in the JSP page using out.println() or JSTL tags. Updated view returned to browser.
SYSTEM ADVANTAGES
🗄️
Centralized Data Storage
All motor specs, rocket configs, and launch records stored in one MySQL database — no file-based storage needed.
🌐
Web-Based Access
Any operator can access the system via browser without installing the Java console application locally.
📊
Launch History Analytics
All previous flights are stored permanently, enabling comparison of apogee across A/B/C motor classes over time.
🔒
Role-Based Security
Session management ensures only authenticated admins can modify motor and rocket data while operators can only view records.
